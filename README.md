# Family Planner Telegram Bot

Telegram-бот для семейного планирования: календарь, задачи, покупки, обязанности, меню, финансы, заметки, напоминания и AI-помощник через OpenAI API.

## Возможности MVP

- Быстрое меню Telegram вместо запоминания команд.
- "Сегодня" и "Неделя" показывают семейные события, задачи, покупки, вишлисты и напоминания.
- Кнопки "Событие", "Задача", "Покупки", "Вишлист", "Напоминание" открывают пошаговые конструкторы.
- "Напоминание" создает персональное напоминание.
- Если кто-то добавляет событие, бронь, задачу, покупку, маркетплейс, вишлист, финансовую запись, меню, заметку или напоминание, бот отправляет уведомление в семейный чат.
- Фоновая проверка напоминаний отправляет уведомления в Telegram.

## Установка

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
```

Заполните `.env`:

```env
TELEGRAM_BOT_TOKEN=токен_от_BotFather
OPENAI_API_KEY=ключ_OpenAI
OPENAI_MODEL=gpt-5.2
ALLOWED_CHAT_IDS=
ALLOWED_USER_IDS=
```

`OPENAI_API_KEY` можно оставить пустым, тогда планер будет работать, а кнопка "AI" сообщит, что AI не настроен.

## Запуск

```powershell
python -m family_planner_bot
```

## Быстрое меню

После `/start` бот показывает кнопки:

```text
☀️ Сегодня | 📅 Неделя
🗓️ Событие | ✅ Задача | 🛒 Покупки
🎁 Вишлист | 🔔 Напоминание
```

Нажмите кнопку и заполните конструктор по шагам. Для необязательных полей отправьте `-`.

Примеры ответов в конструкторах:

```text
Событие:
1. завтра 18:30
2. Мама
3. врач
4. health
5. -

Задача:
1. до пятницы
2. Папа
3. оплатить ЖКУ
4. finance
5. monthly

Покупки:
1. молоко
2. 2 л

Вишлист:
1. Мама
2. массажер для спины
3. день рождения
4. ссылка или комментарий

Напоминание:
1. завтра 09:00
2. Мама
3. взять документы в школу
```

Формат с `|` остался только в запасных командах, чтобы быстро вводить записи одной строкой.

## Запасные команды

```text
/start
/help
/members
/member Мама parent #ff6b6b
/event завтра 18:30 | Мама | врач | health
/booking суббота 15:00 | семья | столик в кафе | 4 человека
/task до пятницы | Папа | оплатить ЖКУ | finance
/buy молоко 2 л
/market WB | кроссовки ребенку | 3200 | ссылка
/wish Мама | массажер для спины | день рождения | ссылка
/expense завтра | семья | 4500 | ЖКУ | коммунальные платежи
/menu понедельник | ужин | паста и салат
/note идея: купить билеты на отпуск
/remind завтра 09:00 | Мама | взять документы в школу
/today
/week
/ask Что у нас завтра и что лучше подготовить вечером?
```

## Повторяющиеся события и задачи

Для `/event` и `/task` можно добавить повтор в конце:

```text
/event вторник 17:00 | Ребенок | футбол | секции | weekly
/task завтра 20:00 | Папа | вынести мусор | дом | daily
```

Поддерживаются `daily`, `weekly`, `monthly`.

## Где хранятся данные

По умолчанию база SQLite создается в `family_planner.sqlite3` рядом с проектом. Путь можно поменять через `DATABASE_PATH`.

## Доступ

Если `ALLOWED_CHAT_IDS` и `ALLOWED_USER_IDS` пустые, бот доступен всем, кто может ему написать.

Чтобы закрыть доступ:

1. Напишите боту `/id` из нужного семейного чата или личного диалога.
2. Скопируйте `chat_id` или `user_id` в `.env`.
3. Перезапустите бота.

Пример:

```env
ALLOWED_CHAT_IDS=-1001234567890
ALLOWED_USER_IDS=111111111,222222222
```

Команда `/id` остается доступной всем, чтобы новый член семьи мог прислать свой ID владельцу бота.

## Члены семьи и упоминания

Чтобы бот предлагал выбрать человека кнопкой в конструкторах задач, событий, вишлистов и напоминаний, каждый член семьи должен зарегистрироваться в нужном чате:

```text
/join Мама
```

Если имя не указать, бот возьмет имя из Telegram:

```text
/join
```

После этого на шаге "кому" бот покажет inline-кнопки с членами семьи. Если у человека есть Telegram user_id, уведомления будут содержать кликабельное упоминание.

## OpenAI

AI-помощник использует OpenAI Responses API: бот передает краткий снимок семейного планера и вопрос пользователя. Документация OpenAI рекомендует Responses API как основной интерфейс для генерации ответов и работы с моделями.
